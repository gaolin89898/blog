# 2026-04-09 安装 InputLeap 记录

## 问题概览
今天安装/使用 **InputLeap** 过程中主要遇到两类问题：

1. **网络问题**：系统的代理/网络配置影响了软件依赖下载与访问。
2. **VM 影响联通**：启用/配置虚拟机（VMware 等）后，虚拟网卡与 NAT/桥接设置影响了两端网络互通，导致两边无法正常联通。

---

## 1) 网络问题（代理/访问失败）
- 现象：访问某些站点/下载依赖失败、TLS 连接异常、仓库更新报错等。
- 处理思路：
  - 检查环境变量代理：`http_proxy / https_proxy / all_proxy / no_proxy`
  - 必要时临时关闭代理再进行安装/更新。
  - 确认目标域名是否需要加入 `NO_PROXY` 直连列表。

---

## 2) VM 影响两边联通（虚拟网卡/NAT/桥接）
- 现象：InputLeap 需要两台设备互相连通，但开启 VM 后网络路径/路由变化，导致互 ping 不通或端口不通。
- 常见原因：
  - VMware 创建的虚拟网卡（如 vmnet8/vmnet1）改变了路由优先级。
  - NAT/Host-only/桥接模式选择不当。
  - 防火墙或端口转发未放行。
- 处理思路：
  - 检查宿主机与对端的 IP、网段、路由表是否被虚拟网卡“抢走”。
  - 视需求切换 VM 网络模式：桥接（更像同一局域网）、NAT、仅主机。
  - 临时关闭 VM 网络/虚拟网卡验证是否恢复联通。

---

## 备注
后续如再次遇到问题，优先：
1) 确认两端 IP/网段与路由
2) 确认代理/防火墙
3) 再检查 InputLeap 端口与服务状态
