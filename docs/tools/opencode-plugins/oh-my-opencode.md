# Oh My OpenCode 安装配置

## 简介

**Oh My OpenCode** 是 OpenCode 的专用编排层（Orchestration Layer），提供开箱即用的 AI 智能体、工作流钩子和自动化能力。

### 核心特性

- 🤖 **多智能体编排** - 模块化工作流，为不同任务分配专业智能体并行工作
- ⚡ **开箱即用** - 内置智能体、钩子、MCP、LSP 支持，安装即用
- 🏗️ **构建流程感知** - 理解复杂仓库结构、构建系统和多文件夹设置
- 🔌 **MCP 集成** - 支持 Model Context Protocol 工具集成

## 安装方法

### 前置要求

- 已安装 OpenCode CLI
- Node.js 18+ 

### 安装步骤

```bash
# 通过 npm 安装
npm install -g oh-my-opencode

# 或通过 OpenCode 插件系统安装
opencode plugins install oh-my-opencode
```

### 初始化配置

```bash
# 初始化 Oh My OpenCode 配置
opencode init

# 或在项目中初始化
opencode init --project
```

## 配置说明

### 配置文件位置

初始化完成后会生成配置文件，路径会在命令输出中提示。项目级初始化会在项目目录内生成配置，便于团队共享与版本管理。

### 常见配置项

- 智能体与角色定义
- 工具与 MCP 接入
- 工作流钩子与触发条件
- 项目级规则与忽略目录

## 快速开始

1. 选择一个仓库作为试点项目
2. 初始化并启用默认智能体
3. 运行一次完整的任务流，验证生成质量
4. 逐步扩展到更多任务类型

## 典型场景

### 项目初始化

自动生成项目结构、基础配置与依赖清单。

### 功能开发

从需求描述生成模块代码，并同步更新相关文件。

### 代码审阅

提供变更摘要、风险点提示与改进建议。

## 使用建议

- 优先在小范围任务中验证流程
- 保持配置与约定一致，提升团队协作效率
- 对关键改动进行人工复核与测试验证

## 常见问题

### 初始化失败

检查 CLI 版本与运行权限，必要时重新安装。

### 智能体不生效

确认配置已加载并启用，必要时重新初始化项目。
